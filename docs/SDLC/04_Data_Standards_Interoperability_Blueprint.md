# Data Standards & Interoperability Blueprint

## 1. Introduction
This blueprint dictates how Cadence Clinical aligns with global data standards, ensuring seamless exchange of clinical trial data.

## 2. CDISC Standards Alignment
### 2.1 SDTM (Study Data Tabulation Model)
- eCRF items must be mapped to their corresponding SDTM domain variables during the study design phase.
- Automatic extraction pipelines generate SDTM-compliant datasets from relational EDC data.

### 2.2 ADaM (Analysis Data Model)
- Derived variables and analysis datasets are supported via the export engine, utilizing SDTM source mappings.

### 2.3 CDASH (Clinical Data Acquisition Standards Harmonization)
- The platform provides an extensive library of CDASH-compliant templates for rapid study startup.
- Core safety domains (AE, CM, MH, VS) adhere strictly to CDASH naming conventions and required fields.

## 3. Medical Dictionaries
### 3.1 MedDRA Integration
- Adverse Events (AE) and Medical History (MH) form items can be bound to the MedDRA search widget.
- Supports auto-coding based on verbatim terms, capturing LLT, PT, HLGT, and SOC hierarchies.

### 3.2 WHODrug Integration
- Concomitant Medications (CM) utilize the WHODrug dictionary for ATC classification.

## 4. Data Export Formats
- Supported formats: CDISC ODM-XML (v1.3.2), Dataset-JSON, and traditional flat CSV files.
- Real-time data streaming to external data warehouses is supported via Apache Kafka integrations.

- Derived variables and analysis datasets are supported via the export engine, utilizing SDTM source mappings.

- eCRF items must be mapped to their corresponding SDTM domain variables during the study design phase.
- Automatic extraction pipelines generate SDTM-compliant datasets from relational EDC data.

### 2.2 ADaM (Analysis Data Model)
- Derived variables and analysis datasets are supported via the export engine, utilizing SDTM source mappings.

### 2.3 CDASH (Clinical Data Acquisition Standards Harmonization)
- The platform provides an extensive library of CDASH-compliant templates for rapid study startup.
- Core safety domains (AE, CM, MH, VS) adhere strictly to CDASH naming conventions and required fields.

## 3. Medical Dictionaries
### 3.1 MedDRA Integration
- Adverse Events (AE) and Medical History (MH) form items can be bound to the MedDRA search widget.
- Supports auto-coding based on verbatim terms, capturing LLT, PT, HLGT, and SOC hierarchies.

### 3.2 WHODrug Integration
- Concomitant Medications (CM) utilize the WHODrug dictionary for ATC classification.

## 4. Data Export Formats
- Supported formats: CDISC ODM-XML (v1.3.2), Dataset-JSON, and traditional flat CSV files.

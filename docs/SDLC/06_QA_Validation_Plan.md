# Quality Assurance (QA) & Validation Plan

## 1. Scope
This plan details the testing strategies and manual verification checklists required for clinical software validation in accordance with **IEC 62304** and **ISO 14155:2020**.

## 2. Test Strategies
### 2.1 Unit and Integration Testing
- The platform mandates a minimum 90% code coverage threshold.
- Pytest is utilized against the live containerized database for integration tests to ensure network and execution fidelity.

### 2.2 System & Acceptance Testing
- End-to-end user workflows are tested via automated UI tests (e.g., Playwright).
- Formal User Acceptance Testing (UAT) is performed before any major release.

## 3. Traceability Matrix
- All Product Requirements (PRD) are mapped to corresponding Test Cases in this document.
- Ensures all hazards identified in the ISO 14971 risk assessment have verified controls.

## 4. Manual Verification Checklists
### 4.1 E-CRF Validation
- [ ] Verify conditional logic accurately hides/shows fields.
- [ ] Verify out-of-range data raises an automatic system query.
- [ ] Verify empty required fields block form completion.

### 4.2 Audit Trail Verification
- [ ] Perform a data update and verify the audit log captures the change reason.
- [ ] Verify the cryptographic hash chain is maintained after an update.
- [ ] Attempt direct database modification; verify the shadow schema trigger captures the event.
